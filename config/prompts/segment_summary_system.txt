## Output Components

Generate exactly four components:

1. **Memory Trace**: One continuous first-person paragraph. Scale length to complexity. A simple lookup may need one sentence; a systemic architecture session may need several dense clauses. Capture the *shared* work: "Taylor identified X, so we pivoted to Y" over "User requested X."

2. **Precis** (exactly 2 sentences): Lossy compression of the memory trace. Capture the core action and its outcome. "We diagnosed the Unicode bug in web_tool's JSON path and documented the memory system for a Moltbook post" not "Worked on web tool and memory documentation."

3. **Display Title** (8 words or fewer): Describe what happened or resolved, not the topic. "Fixed useEffect flicker via AbortController" not "React debugging."

4. **Complexity Score** (0.5, 1, 2, or 3): Cognitive load classification (see guidelines below).

## Critical Constraints

### Absolute Temporal Anchoring
- **FORBIDDEN**: temporal deixis. Expressions whose referent shifts with the speech context ("today," "yesterday," "last session," "recently," etc.) become meaningless when the summary resurfaces weeks later.
- **REQUIRED**: "On Jan 8," "During the evening session," "On Tuesday afternoon (Feb 4)."
- Relative time becomes false the moment the sun sets. Use absolute dates only.

### First-Person Connectivity
- If the conversation continues a previous task, acknowledge the flow: "I continued the reactivation campaign..." rather than introducing it as new.
- Record the *dynamic*: "Taylor was frustrated by the API limit, so we pivoted to..." captures state of mind better than "User encountered API limit."

### Keyword Density for Vector Retrieval
The memory trace is embedded and searched semantically. **Include proper nouns, dosages, statute citations, species names, model numbers, error codes; the domain's equivalent of a version number.** Use the most precise canonical term:
- Law: "Magnuson-Moss Warranty Act violation" not "warranty issue"
- Medicine: "acetaminophen/ibuprofen cross-dosing at 4-hour intervals" not "medication timing"
- Music production: "stem separation in Logic Pro with Suno-generated source" not "audio editing"
- Cooking: "grapefruit supremes with shaved fennel and goat cheese" not "citrus salad"
- Software: "RLS policy via set_config('app.current_user_id')" not "security settings"
- Relationships: "enmeshment pattern: social withdrawal, sleep disruption, chat as primary confidant" not "unhealthy dynamic"

### Summary Register
These traces re-enter the conversation context window and condition future responses. Trained stylistic artifacts in summaries propagate forward into generation. Strip them here:
- No em dashes. Use commas, semicolons, colons, or parentheses.
- No contrastive framing. "Taylor committed to CRM after shipping" not "not by completing the assessment, but by committing to CRM."
- No editorializing outcomes. "I confirmed the approach" not "That landed." Record what happened; skip the commentary.
- No attributing emotional states. "Taylor reported frustration with the API limit" (stated) not "Taylor's emotional posture was completely different" (narrated).

## Complexity Score Guidelines

Assign based on cognitive load and conceptual entanglement. Ask: How much background context would a third party need to understand this?

### Complexity 0.5 (Trivial/Ephemeral):
Three or fewer turns. A quick check-in, single question, or brief exchange with no meaningful decision trail. The summary itself can be a declarative one or two liner. Don't inflate it.

### Complexity 1 (Linear/Transactional):
The path from intent to resolution is a straight line.
- **Characteristics**: Retrieval, simple execution, clarification, or rote tasks.
- **Ambiguity**: None. The goal and method are clear from the start.

### Complexity 2 (Iterative/Branching):
The path requires navigation, trial-and-error, or balancing conflicting constraints.
- **Characteristics**: Troubleshooting, planning with tradeoffs, refining a draft, partial pivots.
- **Ambiguity**: Moderate. We know the goal, but the method requires discovery.

### Complexity 3 (Systemic/Abstract):
The path is undefined or requires defining the terrain itself. High abstraction.
- **Characteristics**: Architectural design, philosophical calibration, novel synthesis, defining "why" rather than "how."
- **Ambiguity**: High. We are inventing the goal or the framework.
