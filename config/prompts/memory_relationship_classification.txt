Classify the relationship between a newly extracted memory and an existing memory from the user's long-term memory system.

These memories were identified as potentially related. Your job is to classify HOW they relate — not whether they do. The key question: would linking these memories enable useful reasoning about the user? If knowing one memory would change how you use the other, they should be linked.

RELATIONSHIP TYPES:

**supports** - Reinforces, corroborates, or provides context
- Information that strengthens or validates the existing memory
- Additional evidence for the same conclusion
- Context that makes the existing memory more useful
- Decision test: Does this make the other memory more credible or useful?

**conflicts** - Mutually exclusive or contradictory information
- Different values for the same fact (e.g., "uses Python 3.9" vs "uses Python 3.11")
- Temporal contradictions (same event, different dates/times)
- Contradictory preferences or beliefs
- Factual disagreements that cannot both be true
- Decision test: If one is true, must the other be false?

**supersedes** - Replaces due to temporal progression
- Temporal progression ("switched from X to Y", "now uses Z instead")
- Explicit updates ("changed preference", "moved from A to B")
- New information that makes old information outdated
- Must have clear temporal or progressive relationship
- Decision test: Does temporal progression make the old memory no longer current?

**refines** - Adds detail or nuance
- More specific information about the same topic
- Clarification or elaboration of existing knowledge
- Precision improvements without contradiction
- Decision test: Does this add granularity without changing the core meaning?

**precedes** - Temporal sequence without causation
- Events that happened before, providing timeline context
- Historical background relevant to understanding current state
- Sequence matters but no direct cause-effect relationship
- Decision test: Did this happen before and help explain the timeline?

**contextualizes** - Background information that frames another memory
- Domain knowledge that helps interpret specific memories
- General principles that explain specific behaviors
- Broader context that makes the specific memory more meaningful
- Personal facts that inform decisions in seemingly unrelated areas
- Decision test: Does knowing this memory change how you'd act on the other?

**null** - Genuinely unrelated
- Completely different topics with no conceptual bridge
- No scenario where knowing one would affect use of the other
- Decision test: Could you use each memory fully without ever knowing the other exists?

**Classification Criteria**:

1. **Conceptual Connection**: Do these memories share a person, place, topic, value, constraint, or implication — even across different life domains?
2. **Temporal Analysis**: Check happens_at/expires_at fields for temporal progressions
3. **Factual Consistency**: Identify if both can be simultaneously true
4. **Specificity**: Prefer more specific relationship types over generic
5. **User Perspective**: Would a thoughtful friend see these as connected?

**Decision Process**:

1. If memories describe contradictory facts -> **conflicts**
2. If new memory explicitly updates/replaces old due to temporal progression -> **supersedes**
3. If new memory reinforces or provides evidence for existing -> **supports**
4. If new memory adds detail/nuance to existing -> **refines**
5. If new memory happened before and provides timeline context -> **precedes**
6. If new memory provides background, constraint, or implication that frames existing -> **contextualizes**
7. If genuinely unrelated with no conceptual bridge -> **null**
